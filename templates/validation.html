About Data Validation

Validation Types

  • Completeness - Checks for missing values in critical fields
  • Data Types - Validates that all fields have the correct data types
  • Numeric Ranges - Ensures numeric values are within reasonable ranges
  • Dates - Verifies dates are valid and within acceptable ranges
  • Duplicates - Identifies duplicate property records
  • Cross-Source Consistency - Checks for consistency across different data sources

Validation Status

  • Passed - All validation checks passed
  • Failed - One or more validations failed
  • Warning - Minor issues were detected
  • No Data - No property data found for validation

To run the ETL pipeline with validation:
python main.py --sources all --validate-only
